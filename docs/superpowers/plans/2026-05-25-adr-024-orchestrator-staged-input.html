3. Phase A — Schema delta

  • A.1 Create packages/orchestrator/src/migrations/postgres/002_input_refs.sql: 
    ALTER TABLE orchestrator.step_results
  ADD COLUMN IF NOT EXISTS payload_kind TEXT;

CREATE UNIQUE INDEX IF NOT EXISTS step_results_input_idempotency_idx
  ON orchestrator.step_results (run_id, step_id, unit_index, payload_kind)
  WHERE payload_kind IS NOT NULL;
    Partial index leaves legacy untyped rows untouched.
  • A.2 Update packages/orchestrator/package.json db:push if needed to apply 002 alongside 001.

4. Phase B — writeStepResult idempotency

  • B.1 Extend WriteStepResultArgs in packages/orchestrator/src/postgres.ts with optional payloadKind.
  • B.2 When payloadKind is set, INSERT with ON CONFLICT (run_id, step_id, unit_index, payload_kind) DO NOTHING RETURNING id; fall back to SELECT on conflict. When unset, keep ADR-022 behaviour (fresh ULID).
  • B.3 Failing tests in postgres.test.ts:
    • Round-trip with payloadKind: "input": insert returns id; repeat returns same id; payload unchanged.
    • Without payloadKind: behaviour unchanged (fresh ULID per call).
    • Re-write with same key but different payload: existing row's payload wins (idempotency over upsert).
  • B.4 pnpm --filter @engineerdad/orchestrator test all green.
  • B.5 Commit feat(orchestrator): writeStepResult idempotencyKey via payload_kind tag (ADR-024 Phase B).

6. Phase D — Refactor P1-fanout

  • D.1 Locate hook-bank inlining in packages/orchestrator/src/stages/produce.ts P1-fanout.
  • D.2 Refactor P1's build to async using ctx.stageInput: 
    build: async (run, ctx): Promise<Step> => {
  const scripts = approvedScriptsFor(run);
  const units = await Promise.all(scripts.map(async (script, i) => {
    const hookBank = hookBankForBrief(run, script.briefId);
    const inputRef = await ctx.stageInput(i, {
      scriptId: script.id, briefId: script.briefId, hookBank,
    });
    return { spawnPrompt: thinPromptWithRef(run.runId, inputRef) };
  }));
  return { kind: "fanout", stepId: "P1-fanout", worker: "creative-director", units };
}
