The produce stage spawns creative-director, a Phase 5 deliverable — so Phase 2 proves plan/verify/advance + the /loop conductor end-to-end against a fixture stage (a real 2-step stage that spawns a trivial worker — no throwaway agent). The real produceStage definition + verifyProduce are written and unit-tested as pure data/functions; their live creative-director spawn is activated in Phase 5 by adding produceStage to the MCP's registry. Phase 2's MCP ships with the fixture stage as its registry, so /loop runs end-to-end and is demonstrable today.

Goal: A typed, deterministic run-state machine (packages/orchestrator), a thin MCP wrapper exposing plan/verify/advance/status, and the domain-agnostic /loop conductor command — proven end-to-end on a fixture stage.

Architecture: Logic in packages/orchestrator (pure, unit-tested); transport in mcp-servers/orchestrator (thin stdio wrapper, ADR-005 thin-adapter). State in data/engineerdad.sqlite via node:sqlite. plan is a pure function of persisted run state — the resumability property.

Tech stack: TypeScript (ESM), node:sqlite (DatabaseSync), @modelcontextprotocol/sdk ^1.18, Zod, vitest. Execution: inline, TDD, one commit per task.

1.2 — Core types (types.ts)

export type RunStage = "fixture" | "produce" | "done";
export type RunStatus = "active" | "awaiting_gate" | "blocked" | "done";
export type StepStatus = "pending" | "done" | "failed";

export interface RunStepState {
  stepId: string;
  stage: string;
  status: StepStatus;
  result: unknown;
  problems: string[];
  attempts: number;
}

/** The full run state — the pure input to `plan`. */
export interface RunState {
  runId: string;
  stage: RunStage;
  status: RunStatus;
  params: Record<string, unknown>;
  steps: RunStepState[];
}

export type StepKind = "spawn" | "fanout" | "write" | "gate" | "done" | "halt";

/** A concrete next action, returned by `plan` for the conductor to execute. */
export type Step =
  | { kind: "spawn";  stepId: string; agent: string; spawnPrompt: string }
  | { kind: "fanout"; stepId: string; worker: string; units: { spawnPrompt: string }[] }
  | { kind: "write";  stepId: string; calls: { tool: string; args: unknown }[] }
  | { kind: "gate";   stepId: string; gate: string; message: string }
  | { kind: "done";   message: string }
  | { kind: "halt";   stepId: string; reason: string };

export interface VerifyResult { ok: boolean; problems: string[]; }

/** Declarative spec for one step of a stage. `build` turns run state into a concrete Step. */
export interface StepSpec {
  id: string;
  kind: "spawn" | "fanout" | "write" | "gate";
  build: (run: RunState) => Step;
  verify?: (run: RunState, result: unknown) => VerifyResult;
}

export interface StageDefinition {
  id: RunStage;
  steps: StepSpec[];
}

1.3 — Engine surface (engine.ts)

/** Continue a run, or mint a new one when `runId` is absent. Pure function of persisted state. */
plan(input: { runId?: string; args?: string }, registry: StageDefinition[]): Step

/** Run the StepSpec's verifier (default: result is non-null and not an {error}). */
verify(runId: string, stepId: string, result: unknown, registry: StageDefinition[]): VerifyResult

/** Mark the step done + persist its result; advance the stage when its last step completes. */
advance(runId: string, stepId: string, result: unknown, registry: StageDefinition[]):
  { stage: RunStage; status: RunStatus }

plan algorithm: (1) no runId → mint run_<unix>, parse args into params, createRun at the registry's first stage; (2) load RunState; (3) in the current stage, find the first StepSpec with no done run-step → build + return it; (4) all steps done → advance to the next registry stage and return its first step; (5) no next stage → return { kind: "done" }. Resumable by construction — step (3) is a diff against persisted state, never a counter.

1.4 — verifyProduce (pure; verifiers/verify-produce.ts)

Phase 2 ships the pure function — inputs passed as arrays, no Notion I/O. The Notion-reading wrapper that feeds it is Phase 5. This is the design centerpiece (the old "Step 7.0 assertion").

interface ProduceScript  { id: string; }
interface ProduceVariant {
  id: string; scriptId: string; format: string; aspect: string;
  channels: string[]; assetFiles: { url: string; sha256: string }[];
  metaSpecComplete: boolean; organicSpecComplete: boolean;
  complianceCheck: boolean; estCostMyr: number;
}

export function verifyProduce(
  scripts: ProduceScript[],
  variants: ProduceVariant[],
  reportedTotalMyr: number,
  renderWorkersRan: number,
): VerifyResult {
  const problems: string[] = [];
  const FORMAT_MATRIX = 5;                 // Reel, Feed, YT-Long, Carousel x2

  for (const s of scripts) {
    const n = variants.filter((v) => v.scriptId === s.id).length;
    if (n !== FORMAT_MATRIX) problems.push(`script ${s.id}: ${n}/${FORMAT_MATRIX} variants`);
  }
  const isStatic = (v: ProduceVariant) => v.format === "Feed" || v.format === "Carousel";
  for (const v of variants.filter(isStatic)) {
    if (v.assetFiles.length === 0) problems.push(`variant ${v.id}: Asset Files empty`);
  }
  for (const v of variants) {
    const meta = v.channels.some((c) => c.startsWith("Meta"));
    if (meta && !v.metaSpecComplete) problems.push(`variant ${v.id}: Meta spec incomplete`);
    if (v.channels.includes("Meta-organic") && !v.organicSpecComplete)
      problems.push(`variant ${v.id}: organic spec incomplete`);
    if (!v.complianceCheck) problems.push(`variant ${v.id}: compliance scan did not pass`);
  }
  const sum = variants.reduce((a, v) => a + v.estCostMyr, 0);
  if (sum !== reportedTotalMyr) problems.push("totals.cost drifts from per-variant sum");
  const staticN = variants.filter(isStatic).length;
  if (staticN > 0 && renderWorkersRan === 0)
    problems.push(`${staticN} static variants, 0 render workers ran`);

  return { ok: problems.length === 0, problems };
}

1.5 — The /loop conductor (.claude/commands/loop.md body)

Repeat until a step says STOP:
1. step = mcp__orchestrator__plan   (first call { args:"$ARGUMENTS" }; then { runId })
2. by step.kind:
   spawn  -> Task(step.agent, step.spawnPrompt)            verbatim
   fanout -> Task x N general-purpose workers, parallel    verbatim
   write  -> execute step.calls against the named MCP      verbatim
   gate / done / halt -> print message, STOP
3. spawn|fanout|write -> v = mcp__orchestrator__verify({ runId, stepId, result })
4. v.ok            -> mcp__orchestrator__advance(...) ; goto 1
   v.ok === false  -> [judgment] transient: retry once. structural: print problems, STOP.

Domain-agnostic — no marketing concept appears. status.md calls mcp__orchestrator__status and prints the run table.
