CLAUDE.md Hardening

Design spec  ·  2026-05-25  ·  engineerdad-marketing

1 · Goal

Harden the repo's orientation files so Claude Code (and human developers) always have an accurate, concise picture and so the files stay current automatically. The four principles driving every change:

  1. No redundancy — each fact lives in one place only.
  2. CLAUDE.md stays thin — every line costs session tokens; cut ruthlessly.
  3. ARCHITECTURE.md is the SSOT for the system as built; CLAUDE.md links to it.
  4. Maintenance is explicit — CLAUDE.md tells Claude when to update which file.

3.6 New "Maintenance" section add

Three explicit triggers — tells Claude when to update which file without ambiguity:

## Maintenance

- **Architectural change** (new MCP server, stage, agent, or storage layout) →
  update `ARCHITECTURE.md`; keep this file's big-picture summary in sync.
- **Bug or enhancement uncovered** → open a new entry in `TASKS.md`.
- **Work ships / ticket closes** → close the entry in `TASKS.md`, refresh the
  Status header.
