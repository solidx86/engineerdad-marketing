Changes since last review

The substrate moves from SQLite (committed-to-git) to Postgres 16-alpine in Docker compose, with the volume bind-mounted to ./data/postgres/ (gitignored). Three cascading effects:

  1. Task 2 ↔ Task 3 swapped & rewritten. Task 2 is now "Switch foundation to Postgres + Docker compose" — it amends Task 1's SQLite scaffold (deps, db.ts, drizzle.config.ts), brings up the container, and adds store:up/down/wipe/push/logs root scripts. Task 3 is now "Port 8 entity schemas to Drizzle (pgTable)" — uses Postgres-native types (uuid, jsonb, timestamptz, boolean).
  2. Test DB pattern reshaped. Unit tests (Task 5, Task 8) run against the live engineerdad_test database with TRUNCATE ... CASCADE between cases. Playwright (Task 14) uses DROP SCHEMA public CASCADE; CREATE SCHEMA public; drizzle-kit push as the per-spec reset.
  3. Setup gains Docker. First-time setup: install Docker → pnpm install → pnpm -r build → pnpm store:up && pnpm store:push → pnpm review. data/postgres/ is gitignored — clone has no data; cold-start is the only way in.

ADR-021 + the spec HTML both updated. Tasks 4 (filters), 6 (compliance), 7 (barrel), 9, 10, 11, 13, 15, 17, 19 are unchanged. Tasks 12 + 16 + 18 have small adjustments (Next config drops serverExternalPackages; README setup adds Docker; cold-start walk adds pnpm store:up).

Goal & architecture

Goal: Retire Notion as the OS's data + human-gate substrate. Ship a typed Postgres store running in a Docker container, a cap-honouring MCP wrapper, and a single-user Next.js review UI so a fresh /loop-once walks cold-start to HG3 without any tool result approaching the MCP wire cap.

Architecture: Drizzle ORM (pgTable) over postgres.js against a Postgres 16-alpine container; bind-mount data volume to ./data/postgres/ (gitignored). Five-tool MCP surface (query returns IDs only, get per row, create/update/set_status) so bulk content never crosses the conductor boundary. Next.js 15 App Router with server actions over the same packages/store library. Compliance scanner moves from the Notion MCP handler into packages/store/src/crud.ts. Orchestrator run-state (runs/run_steps) stays in data/engineerdad.sqlite — two stores side-by-side, each fit for purpose.
