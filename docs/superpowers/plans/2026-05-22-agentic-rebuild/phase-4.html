Four decisions baked in 1 — tracking + analytics = pure write-step chains. Both agents are fixed pipelines of MCP calls. As stages they are write steps with verifiers; tracking's halt-the-loop becomes a verify-fail STOP — no new machinery.
2 — detectFatigue ships as a pure function; the fatigue artifact is computed at consumption time. The analytics stage gathers (pull, rank, decay, angle); detectFatigue is a tested pure function applied by the build of the step that consumes analytics output — Brain's reasoning step, Phase 6 — exactly as planDistribution is computed in distribute's D2 build.
3 — assignSchedule is a pure function in shared/derive; schedule is a 2-step stage after HG3. Slot assignment, MYT→UTC, the publish window — zero open decision space (design §7).
4 — registry timing (C1). The three stages ship as tested definitions; the live registry stays fixture-only. Phase 5 wires the ordered live registry and deletes the replaced agents.

Goal: Three new orchestrator stages — tracking, analytics, schedule — plus assignSchedule and detectFatigue as pure tested functions, and the organic-FB routing folded into planDistribution (deferred from Phase 3).

Architecture: tracking and analytics are deterministic write-step chains. schedule runs after HG3 and stamps Scheduled For via assignSchedule. All build against Phase 2's frozen engine; no engine change.

Tech stack: TypeScript (ESM), node:sqlite, vitest. New deps: none. Execution: TBD — this doc is for review first.

1.2 — assignSchedule (shared/derive/schedule.ts)

export interface ScheduleInput {
  variantId: string;
  format: string;          // Reel | Feed | Carousel | YT-Long | YT-Short
  channels: string[];      // Meta-organic ⇒ organic branch; Meta-paid ⇒ paid branch
}
export interface OrganicSlot  { variantId: string; scheduledForUtc: string; } // ISO
export interface PaidFlight   { variantId: string; flightStartUtc: string; flightEndUtc: string; }
export interface ScheduleResult {
  organic: OrganicSlot[];
  paid: PaidFlight[];
  problems: string[];      // a slot outside the publish window, etc.
}

/** Organic: assign each Meta-organic variant the next free cadence slot from
 *  `now`, convert MYT→UTC, reject anything outside [now+10min, now+75d].
 *  Paid: a flight of `durationDays` from `now`. Pure — `now` is injected. */
export function assignSchedule(
  variants: ScheduleInput[],
  opts: { now: Date; durationDays?: number },
): ScheduleResult

Cadence table (MYT, from the retired distribution §4d): Image/Feed → Mon 20:00, Wed 20:00, Fri 18:00; Carousel → Tue 19:00; Reel → Thu 19:00. Slots fill in variant order; a week rolls over when its slots are spent. MYT = UTC+8.

1.3 — detectFatigue (analytics/fatigue.ts)

export interface DecayCurve { adId: string; points: { date: string; cpa: number }[]; }
export interface FatigueRow {
  adId: string; baselineCpaMyr: number; recentCpaMyr: number; deltaPct: number;
}
/** §12.5 sunset: an ad is fatiguing when its mean CPA over the last 3 days is
 *  >25% above baseline (median of the first 7 points, or the first half if
 *  the curve is shorter). Pure. */
export function detectFatigue(curves: DecayCurve[]): FatigueRow[]
