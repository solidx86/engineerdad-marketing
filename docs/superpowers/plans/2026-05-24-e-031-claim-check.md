# E-031 — Claim-Check for worker output transmission — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the Claim-Check (Hohpe EIP) primitive across the `/loop` conductor's transmission path. Workers persist their final output to a new `orchestrator.step_results` Postgres table and return only a `{stepResultId}` ref (~50 bytes). The conductor carries refs verbatim. The orchestrator MCP dereferences server-side before calling `engine.verify` / `engine.advance`. Engine, state.ts, verifiers, and downstream `stepResult<T>()` consumers are unchanged. Doctrine becomes structurally enforced: the conductor cannot reshape what it does not hold.

**Architecture:** A new `orchestrator` schema in the existing Postgres container (alongside the entity tables in `public`). `step_results` columns: `id` (TEXT with `sr_` ULID prefix), `run_id`, `step_id`, `unit_index` (nullable for spawn), `payload` (JSONB), `size_bytes`, `created_at`. Two new MCP tools on `mcp-servers/orchestrator/` — `write_step_result` (worker persistence) and `read_step_result` (auditable opt-in transparency for the conductor or any agent that needs to reason over a worker's output). `verify`/`advance` handlers detect ref-shaped results (`{stepResultId}` or `[{stepResultId}, ...]`) and resolve via a new `resolve.ts` helper before calling into the engine. Per-kind enforcement at the MCP layer rejects inline payloads for `spawn`/`fanout` steps. All four worker agents (brain, brief-writer, content-writer, creative-director) update their emit points to write step_result before returning. `loop.md` §2 step-kind doctrine simplifies: the conductor just passes refs verbatim.

**Tech stack:** TypeScript, pnpm workspaces, Vitest, postgres.js, Drizzle (raw SQL migration — no new schema in `packages/store`'s drizzle config), `ulid` npm package, Zod. No new top-level dependencies in the workspace beyond `ulid` and a postgres.js connection in `packages/orchestrator`.

**Spec:** `docs/superpowers/specs/2026-05-24-e-031-claim-check-design.html`
**Doctrine:** `docs/decisions/022-claim-check-worker-output.md` (Draft)
**Branch:** `feat/e-031-claim-check` (new, off `feat/agentic-rebuild`)

**Pinned forks (from SPEC §6):**
- Substrate: **Postgres**, new `orchestrator` schema in the existing `engineerdad` database.
- Write tool: new `mcp__orchestrator__write_step_result` (bypasses entity-store compliance scanner).
- Read tool: new `mcp__orchestrator__read_step_result` (auditable opt-in transparency).
- Dereference site: **MCP layer**, in `mcp-servers/orchestrator/src/index.ts`. Engine + state.ts untouched.
- ID format: ULID with `sr_` prefix (e.g. `sr_01HZA9...`).
- Back-compat: **hard-cut** for `spawn` + `fanout` step kinds; `write` + `gate` keep inline.
- Rollout: all four worker agents at once on the feature branch.
- Naming: "Claim-Check" anchored to Hohpe's EIP; Google ADK's "handle pattern" as alt term.

**Out of scope (deliberately):** `runs`/`run_steps` migration to Postgres (E-030); TTL/sweep job for old step_results (provisional E-033); render-worker integration audit (audit during Phase D, fold in only if needed); per-worker token cost reduction (E-032); changing worker payload shapes.

- [ ] **Step 2:** Create `packages/orchestrator/src/postgres.ts`:
  ```ts
  import postgres from "postgres";
  import { newStepResultId } from "./ulid.js";

  type Sql = ReturnType<typeof postgres>;

  let cached: Sql | undefined;

  function client(): Sql {
    if (cached) return cached;
    const url = process.env.DATABASE_URL;
    if (!url) throw new Error("DATABASE_URL not set; orchestrator step_results requires Postgres");
    cached = postgres(url, { prepare: false });
    return cached;
  }

  export interface WriteStepResultArgs {
    runId: string;
    stepId: string;
    unitIndex?: number;
    payload: unknown;
  }

  /** Insert one row; return the generated stepResultId. */
  export async function writeStepResult(args: WriteStepResultArgs): Promise<string> {
    const id = newStepResultId();
    const payloadText = JSON.stringify(args.payload);
    const sizeBytes = Buffer.byteLength(payloadText, "utf8");
    const sql = client();
    await sql`
      INSERT INTO orchestrator.step_results
        (id, run_id, step_id, unit_index, payload, size_bytes)
      VALUES
        (${id}, ${args.runId}, ${args.stepId},
         ${args.unitIndex ?? null}, ${payloadText}::jsonb, ${sizeBytes})
    `;
    return id;
  }

  /** Load a payload by stepResultId. Throws StepResultNotFoundError if missing. */
  export async function loadPayload(stepResultId: string): Promise<unknown> {
    const sql = client();
    const rows = await sql<{ payload: unknown }[]>`
      SELECT payload FROM orchestrator.step_results WHERE id = ${stepResultId}
    `;
    if (rows.length === 0) {
      throw new StepResultNotFoundError(stepResultId);
    }
    return rows[0]!.payload;
  }

  export class StepResultNotFoundError extends Error {
    readonly stepResultId: string;
    constructor(stepResultId: string) {
      super(`step_result not found: ${stepResultId}`);
      this.name = "StepResultNotFoundError";
      this.stepResultId = stepResultId;
    }
  }

  /** Test-only: close the cached connection. */
  export async function closePostgres(): Promise<void> {
    if (cached) {
      await cached.end({ timeout: 5 });
      cached = undefined;
    }
  }
  ```

### Task 6: Implement the resolve helper

- [ ] **Step 1:** Create `mcp-servers/orchestrator/src/resolve.ts`:
  ```ts
  import { loadPayload } from "@engineerdad/orchestrator/postgres";

  export const isRef = (x: unknown): x is { stepResultId: string } =>
    typeof x === "object" && x !== null
      && "stepResultId" in x
      && typeof (x as { stepResultId: unknown }).stepResultId === "string"
      && (x as { stepResultId: string }).stepResultId.startsWith("sr_");

  /**
   * Resolve a Verify/Advance `result` arg into the shape engine.verify expects.
   * - { stepResultId } (spawn)       → single payload
   * - [{ stepResultId }, ...] (fanout) → array of payloads
   * - anything else                   → pass through (write / gate)
   */
  export async function resolveRefs(result: unknown): Promise<unknown> {
    if (isRef(result)) return loadPayload(result.stepResultId);
    if (Array.isArray(result) && result.length > 0 && result.every(isRef)) {
      return Promise.all(result.map((r) => loadPayload(r.stepResultId)));
    }
    return result;
  }
  ```
- [ ] **Step 2:** Ensure `@engineerdad/orchestrator` exports `postgres.ts` symbols via `packages/orchestrator/src/index.ts`. Add the re-export if missing.
- [ ] **Step 3:** Write `mcp-servers/orchestrator/src/resolve.test.ts`:
  - Spawn shape: pre-insert a row; `resolveRefs({stepResultId: id})` → returns the payload.
  - Fanout shape: pre-insert N rows; `resolveRefs([{stepResultId: id1}, ...])` → returns array of N payloads in order.
  - Inline pass-through: `resolveRefs("plain string")` returns the string; same for `{rows: [...]}` and `[{foo: 1}, {bar: 2}]` (mixed-non-ref array).
  - Missing ref: `resolveRefs({stepResultId: "sr_doesnotexist..."})` throws `StepResultNotFoundError`.
  - Empty array: `resolveRefs([])` passes through (no ref-array detection on empty).
  - Mixed array (some refs, some not): passes through (we only treat *every-element-is-ref* as fanout).

  ```ts
  // Looks up the step's kind in LIVE_REGISTRY; rejects inline for spawn/fanout.
  export function enforceKind(stepId: string, result: unknown, registry: StageDefinition[]): void {
    const spec = findStepSpec(registry, stepId);
    if (!spec) throw new Error(`unknown stepId: ${stepId}`);
    const kind = spec.kind; // "spawn" | "fanout" | "write" | "gate"

    if (kind === "spawn" && !isRef(result)) {
      throw new ContractError(`spawn step ${stepId} requires a {stepResultId} ref, got inline payload`);
    }
    if (kind === "fanout") {
      if (!Array.isArray(result) || !result.every(isRef)) {
        throw new ContractError(`fanout step ${stepId} requires [{stepResultId}, ...] array, got non-ref shape`);
      }
    }
    // write + gate: inline is fine; no check needed
  }

  export class ContractError extends Error {
    constructor(msg: string) { super(msg); this.name = "ContractError"; }
  }
  ```
