B-023 — Experiment cell→variant join (angle-only)

2026-05-28 · feat/brain-experiment-params · supersedes nothing · scoped follow-up to Brain Initiative Phase 0

1. Context

Brain Initiative Phase 0 wired experimentParams from the Decision Memo into X2-design, so run_1779895374 is the first run whose experiment stage actually executes instead of taking the legitimate-skip path. The first such walk surfaced two bugs:

  • B-022 (fixed in this branch, not yet committed) — designCellsOf cast the experiment MCP's snake_case cell_id/factor_levels output to the orchestrator's camelCase DesignCell interface without translating, so downstream Object.entries(cell.factorLevels) threw "Cannot convert undefined or null to object."
  • B-023 (this spec) — mapCellsToVariants joins cells to variants by variant.factorTags[k] === cell.factorLevels[k], but nothing in the system populates factorTags. The store schema doesn't have the column, P3-persist doesn't write it, and X1-query doesn't project it. Cells design correctly (3 cells, 70/10/10 split), but every cell ends up with empty variantPageIds, which makes distribute's cellIdFor(variantId) return null for every Meta-paid variant → all 10 variants get marked skipped with skipReason: "no cellId".

This spec closes B-023 with the smallest surgery that unblocks the end-to-end walk: a join-at-X1-query-time approach, scoped to a single factor (angle).

Scope discipline. Brain currently emits only angle as a factor. format, persona, hook_register, and language are out of scope for this spec — they're plausible future factors but each has its own structural implications (hook_register needs a CD prompt change + new variant column; language needs distribute restructure). Future generalization tracked separately: E-042 (angle + format + persona via the same projection pattern) and E-043 (hook_register + language).

2. Architecture

Resolve variant.angle at X1-query time by reading Scripts and Briefs alongside CreativeVariants, then walking variant.script → script.brief → brief.{angle, budgetBucket} in a pure projection helper. No schema migration. No P3-persist change. No creative-director prompt change. The join stays an experiment-stage concern; mapCellsToVariants and applyAllocation stay untouched pure functions.

  • New helper projectExpVariant(): pure function. Walks the variant→script→brief chain in-memory; degrades gracefully when any link is missing (returns factorTags: {} + budgetBucket: null rather than throwing). 
    function projectExpVariant(
  variantRow: { id: string; script?: string | null },
  scriptsById: Map<string, { brief?: string | null }>,
  briefsById:  Map<string, { angle?: string | null; budgetBucket?: string | null }>,
): ExpVariantRow {
  const script = variantRow.script ? scriptsById.get(variantRow.script) : undefined;
  const brief  = script?.brief ? briefsById.get(script.brief) : undefined;
  const factorTags: Record<string, string> = {};
  if (brief?.angle) factorTags.angle = brief.angle;
  const bucket = brief?.budgetBucket;
  const budgetBucket =
    bucket === "70" || bucket === "20" || bucket === "10" ? bucket : null;
  return { pageId: variantRow.id, factorTags, budgetBucket };
}
  • allocatedCellsFor(): build scriptsById/briefsById lookup maps from x1[3]/x1[4], then project each x1[0] row before passing to mapCellsToVariants.

4. Data flow

X1-query result (after fix):
  [
    [variantRow, ...],     // x1[0] — CreativeVariants {id, script}
    [hypothesisRow, ...],  // x1[1] — Hypotheses
    [experimentRow, ...],  // x1[2] — Experiments
    [scriptRow, ...],      // x1[3] — Scripts {id, brief}        (NEW)
    [briefRow, ...]        // x1[4] — Briefs {id, angle,
                           //                  budgetBucket}      (NEW)
  ]

allocatedCellsFor(run):
  scriptsById = Map(x1[3].map(s => [s.id, s]))
  briefsById  = Map(x1[4].map(b => [b.id, b]))
  variants    = x1[0].map(v => projectExpVariant(v, scriptsById, briefsById))
                // → { pageId, factorTags: {angle}, budgetBucket }
  cells       = designCellsOf(x2[0])
                // → { cellId, factorLevels: {angle} }      // B-022 fix applied
  return applyAllocation(mapCellsToVariants(cells, variants))

For run_1779895374: cell cell_01 (angle: "epf-shortfall-parent-worry") will match all 10 approved variants (all from EPF scripts). Cells cell_02/cell_03 match zero variants → applyAllocation's existing empty-bucket logic redistributes their 20% across the active bucket.

5. Error handling

Degrade-don't-throw. Every missing-link case returns sensible defaults so a partial run state (e.g., script id present but brief deleted) doesn't take the whole stage down. The cells those variants belong to just get empty variantPageIds — observable in the experiment row, audit-trail visible.

8. Risk + rollback

Low risk: the change touches one file (experiment.ts), all logic is pure, and tests cover the projection edge cases. Rollback is the inverse diff. If a future Brain version emits a factor we haven't projected, the variant just won't match cells with that factor — same failure mode as today, observable through the empty variantPageIds in the experiment row, not a crash.
