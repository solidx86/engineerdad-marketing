§1Strategic premise

Organic IG/FB is positioned as trust infrastructure for the paid funnel, not a separate growth channel. The Malaysian parent considering a multi-year unit-trust + PRS decision typically clicks the paid ad → checks IG/FB to verify the consultant is real and active → decides whether to WhatsApp. An abandoned page silently drops 60–80% of high-trust purchase journeys.

Measurement therefore grades organic by paid funnel quality (paid CTR/CPM trend, WhatsApp click-through from paid, direct DMs) and channel-native engagement signals (saves, shares, reach) — not by direct conversion attribution, which is intractable without cross-channel modeling.

Existing audience context: the IG/FB accounts are not cold-starting (few-hundred followers, 30+ historical posts; manual paid running outside this OS). Building automation now is right-sequenced.

§2Cadence

Five posts per week (3 image + 1 carousel + ~0.75 Reel). Reel quota = 3 per rolling 30 days, derived stateless from Organic Published At counts. Most weeks fire 5; on the 4th week of each window, fires 4 when the Reel quota is met.

§5/post-week orchestration

Command signature

/post-week [--week-start=YYYY-MM-DD] [--reel=auto|force|skip] [--resume] [--reset=<variant_id>]

Flow

brain dispatches → organic-planner step (inline in brain.md) 1. Pool query (§5.3) 2. Reel-week decision (§5.4) 3. Select 3 Image + 1 Carousel + (0|1) Reel 4. Assign Organic Scheduled For per slot per §2 5. If Reel selected → spawn media-production phase: reel 6. Set Organic Status = Drafted on all selected rows 7. Return summary: rows queued + pool_short / quota_exceeded warnings ▼ HUMAN GATE — per-post approval in Notion ▼ /distribute --channels=meta-organic (see §7)

Variant selection rules

SELECT FROM CreativeVariants
WHERE  Channels ∋ "Meta-organic"
  AND  "HG3 Status" = "Approved"
  AND  "Organic Status" IS NULL
  AND  "Asset Files" IS NOT EMPTY
GROUP BY Format
PICK:
  3 × Format="Feed 1:1"      ORDER BY run_id DESC,
                                       funnel_stage_priority(TOFU > MOFU > BOFU)
  1 × Format="Carousel 4:5"  same ordering
  1 × Format="Reel 9:16"     IFF reel_week_decision = TRUE

Funnel-stage tie-break favors TOFU/MOFU for organic: organic warms cold-audience trust; BOFU pricing-heavy content runs paid where intent is qualified.

"Is this a Reel week?" — stateless

const reelsLast30Days = countVariants({
  Format: "Reel 9:16",
  "Organic Published At": { gte: now - 30d }
});
const reelThisWeek = reelsLast30Days < 3;  // self-correcting, target 3/mo

User overrides via --reel=force (post even if quota met) or --reel=skip (suppress even if unmet).

§6media-production changes

Step 5.8 (new) — write Meta-organic spec at HG3 time

Invoked alongside existing Step 5.7 (Meta-paid spec) and Step 6 (YouTube spec). For Variants whose Channels includes Meta-organic:

Field writtenConstraints
Organic Caption EN≤2,200 chars; warm-audience storytelling; 1st person; CTA woven into prose; emoji-friendly; 2–3 inline hashtags allowed
Organic Caption BMSame constraints, BM register
Organic Hashtags IG8–15 tags; broad + niche + branded mix
Organic Hashtags FB1–3 tags; brand + topic only

Why HG3 time, not /post-week time

Single review moment: human sees paid + organic + YouTube copy side-by-side at HG3. Voice coherence: caption written with Script + hook fresh in same prompt context. /post-week stays a fast selector — no LLM hot path. Trade-off accepted: cannot reference topical events (evergreen voice is correct for unit-trust authority content).

Phase reel (new) — render Reel MP4 at /post-week selection time

Reels are rendered only on selection (not at HG3) so HeyGen credits are not burned on Variants that never post. Sync path inside /post-week:

1. Read Variant: Script (BM or EN), Funnel Stage, Hook
2. Build HeyGen render spec:
     avatar_id     = $HEYGEN_AVATAR_ID    // one global "Shoo" twin
     voice_id      = $HEYGEN_VOICE_ID     // dual-trained EN+BM
     input_text    = trim(Script.{EN|BM} per Organic Language, ~150 words ≈ 60s)
     language      = Variant.Organic Language    // single source of truth
     aspect_ratio  = "9:16"
3. mcp__heygen__generate_video → returns job_id
4. Stamp Reel HeyGen Job ID on Variant
5. Poll mcp__heygen__get_video_status every 15s (max 5 min)
6. On success: download MP4 → mcp__asset-store__upload → URL
7. Stamp Reel MP4 URL on Variant
8. Return; Reel Variant joins the 4 image/carousel queue

5-minute poll timeout → emit reel_render_pending to Pipeline Notes; /post-week --resume picks it up later. Cost guard: before any HeyGen call, planner counts Reels rendered this calendar month; if ≥4 (one over target), aborts with quota_exceeded.

Script-language guidance for Reels

HeyGen renders one TTS language per video — fluid Manglish (mid-sentence code-switching, particles like lah/lor/ah) sounds rough. Scripts pick one dominant language and sprinkle the other naturally as borrow-words. Matches authority-tone for a unit-trust consultant; Manglish stays reserved for human moments (DM replies, Stories) outside this pipeline.

§7distribution §4d Meta-organic branch

Becomes real (currently stubbed: channel_not_implemented). Pure router per ADR-018 — reads spec verbatim from Notion, never derives.

Filter:
  Channels ∋ "Meta-organic"
  AND Organic Status = "Approved"
  AND Organic Scheduled For IS NOT NULL
  AND (IG Post ID IS NULL OR FB Post ID IS NULL)

For each row:
  1. Read Organic Language (set at HG3, default EN, single source of truth).
     Pick matching caption + matching asset PNG.
     Reels already rendered in Organic Language during /post-week (§6.2).
  2. Compose body per platform:
     <picked caption> + "\n\n" + Organic Hashtags {IG|FB}.join(' ')
     Hashtag arrays are language-agnostic.
  3. IG: mcp__meta-organic__publish_(image|carousel|video)_post
         with scheduled_publish_time = Organic Scheduled For
  4. FB: same
  5. Back-fill IG Post ID + FB Post ID
  6. Both success → Organic Status = Published, stamp Organic Published At
  7. Partial fail → Organic Status = Failed + Pipeline Notes naming the leg

§8.2 HeyGen — register, don't rebuild

  1. Add HeyGen's official MCP to .mcp.json per developers.heygen.com docs.
  2. Env: HEYGEN_API_KEY, HEYGEN_AVATAR_ID, HEYGEN_VOICE_ID.
  3. media-production calls mcp__heygen__generate_video + get_video_status directly.
  4. Fallback: if HeyGen MCP tool shapes don't match, build minimal mcp-servers/heygen-wrapper/ on REST. ~1 day if needed.

One-time HeyGen setup — manual, ~15 min, before first /post-week

Train avatar with mixed EN + BM source video (~2 min total, both languages) — preserves Malaysian-Malay phoneme lip-sync. Train voice clone with mixed EN + BM audio (~90s each) — preserves your personal BM accent vs the generic TTS default. Note resulting avatar_id + voice_id; populate env vars.

§10Safety doctrine — ADR-019 (new)

Decision: schedule-only, never immediate

Unlike paid (status=PAUSED) or YouTube (privacyStatus=unlisted), organic posts have no draft state — once published, they're public to followers and the algorithm. Therefore, four constraints are hard-wired at the mcp-servers/meta-organic/ layer:

  1. No immediate publish path. Every publish_* tool requires scheduled_publish_time ≥ now+10min. No --allow-immediate parameter exists in any tool schema. Refuses with immediate_publish_disabled.
  2. Validate window. scheduled_publish_time ≤ 75 days in future (Meta's hard cap). Refuses with out_of_schedule_window.
  3. Compliance pre-flight. The existing scanner (packages/shared/src/compliance.ts, citing corpus/compliance/{sc-malaysia,fimm,public-mutual}.md) runs on Organic Caption EN/BM before any publish call. Fails closed.
  4. Idempotency key. Each publish call carries (Variant ID, platform) as idem key; double-fires return existing post ID rather than duplicating.

Consequence: a post only lands instantly if the user manually edits Organic Scheduled For to exactly now+10min — and even then Meta queues it server-side. The human always has a 10-minute window to cancel via Meta's UI before it goes live.

§16Acceptance criteria

  1. All three migrations apply cleanly to live Notion + SQLite.
  2. After a /loop-once run, Variants with Channels ∋ Meta-organic have populated Organic Caption EN/BM + Organic Hashtags IG/FB from Step 5.8.
  3. /post-week against a non-empty pool selects 3 images + 1 carousel + (0|1) Reel per §5; Reel renders within 5 min and stamps Reel MP4 URL; all rows flip to Organic Status = Drafted.
  4. After approval, /distribute --channels=meta-organic schedules via Meta native API; back-fills IG Post ID + FB Post ID; sets Organic Status = Published.
  5. ingest_meta_organic_insights populates creative_signals; top_creatives(channel="meta-organic") returns ranked Variants.
  6. A Channel = Meta-organic hypothesis graded by /reflect returns Confirm/Refute/Inconclusive per §9.
  7. Repo-wide pnpm typecheck clean; existing tests pass; new tests cover meta-organic MCP scheduling validation, /post-week selection rules, Reel quota counting, Zod refines.
  8. pnpm sync:agents:check clean.
  9. ADR-019 committed.

§18Decisions made during brainstorming

  1. Reframe accepted: organic = trust infrastructure for paid funnel, not separate growth channel.
  2. Build now, not gated on signal test: user has few-hundred followers + 30+ posts + manual paid history; not cold-starting.
  3. Platforms: IG Business + FB Page only (both via Meta Graph). TikTok / LinkedIn out of scope.
  4. Content source: reuse paid loop's CreativeVariants (no parallel engine).
  5. AI avatar pipeline: HeyGen Creator ($29/mo) + Pay-As-You-Go API (~$10–20/mo). Register HeyGen's official MCP.
  6. Trigger: manual /post-week (not scheduled / not cron).
  7. Approval: per-post (Notion filter on Organic Status = Drafted).
  8. Data model: Option A — extend CreativeVariants (no new DB).
  9. Captions: shared caption + per-platform hashtags (Option Y).
  10. Carousel BOFU channel: corrected to [Meta-paid, Meta-organic].
  11. Scheduling mechanism: Meta native scheduled_publish_time.
  12. Organic copy generation: at HG3 time in media-production Step 5.8.
  13. Notion views: no auto-provisioned filtered views; manual one-time UI grouping.
  14. Pipeline Notes separated from Organic Approval Notes — system vs human.
  15. HeyGen IDs: global env vars (one "Shoo" twin), not per-Variant.
  16. One Reel per Variant in Script's language; mandatory dual-language HeyGen training.
  17. Slice scope: v1 = posting + Meta-organic analytics + multi-channel Hypothesis field. YouTube + Articles + experiment-os multi-channel + cross-channel attribution → separate v1.5/v2 specs.
  18. Cross-channel hypothesis grading: v1 grades each channel independently.
  19. Organic Status: shared state across IG+FB for v1.
  20. Manglish: not viable on current AI TTS — scripts pick one dominant language with sparse borrow-words from the other.
  21. One organic post per Variant in one language — every Script is bilingual but organic publishes in ONE language per Variant. New field Organic Language (single_select EN | BM, default EN) set at HG3; user overrides per Variant. Re-posting same Variant in the other language deferred to v1.5.
