1. Context & problem

The Marketing OS produces four creative formats per Script — Reel, Feed, YT-Long, Carousel — but only Feed and Carousel reach assetFiles[0].url today. Reels are emitted by creative-director at P1-fanout but skip P2-render entirely because STATIC_FORMATS = {Feed, Carousel}. The result: every run produces orphaned Reel variants that fail at HG3 because they have no playable asset.

TASKS.md row E-004 previously filed this as a kie.ai (Veo 3) + media-stitch + ElevenLabs + stock-footage problem. After auditing the brand corpus this spec rejects that scope:

  • corpus/courses/brand-voice.md mandates "Show the math" and "Don't lean on emoji or hype" — generative B-roll is off-brand.
  • 18 chart YAMLs in corpus/data/charts/ are the actual visual register of the brand. static-renderer already turns them into PNG frames for Feed and Carousel.
  • The brand archetype is "Engineer-Sage with the Trader's Reality Check" — Shoo's face + Shoo's charts. Not cinematic generative footage.

The right pipeline for EngineerDad Reels is HeyGen avatar (Shoo's likeness) + static-renderer chart frames stitched via ffmpeg. kie.ai is dropped from scope.

2. Goals & non-goals

Goals

  • Every Reel CreativeUnit produces a playable MP4 in assetFiles[0].url before HG3.
  • Reel rendering is resumable — a worker crash mid-job never orphans a HeyGen invocation (the scar at packages/orchestrator/src/exec.ts:92).
  • The pipeline reuses the existing render-worker pattern (ADR-014); no new orchestrator concepts.
  • Reel assetFiles[0].url flows unchanged into the distribute stage (already video-aware on main).
  • EN-only for V1; EN subtitles burned in for sound-off mobile feeds.

Non-goals

  • BM-language Reels. Filed as a follow-up once EN steady state holds.
  • kie.ai / Veo 3 / cinematic B-roll. Dropped from E-004 scope; revisit only if HG3 rejection data later argues for it.
  • YT-Long (16:9) and YT-Short (9:16-short) Reel variants. Same worker shape, different aspect — filed for after first 5 Reels ship.
  • HeyGen webhook ingestion. Polling-based for now; webhooks require a public inbound endpoint we don't have.
  • Instagram automated publishing. Stays out-of-band via /posting-pack per B-005 until E-024.
  • Cross-run RenderEvents analytics dashboard.

4. Architecture

5. Component boundaries & contracts

5.1 creative-director agent (extended)

Decomposes approved Script → 4 CreativeUnits, including a Reel-aware shotlist. No new tools; prompt extension only.

New Reel shotlist contract (additive to existing CreativeUnit):

{
  format: "Reel",
  aspect: "9:16",
  hook: { en: string, ms: string },              // existing
  shotlistEn: Array<{
    scene: string,
    voiceover: string,                            // ≤ 30 words/scene (pacing budget)
    onScreenText: string,
    chartRef: string | null,                      // existing: chart YAML path
    shotNotes: string,
    // NEW for Reel:
    sceneType: "face" | "chart" | "face-over-chart",
    estimatedSeconds: number,                     // planning hint; corrected by force-alignment
  }>,
  // NEW Reel-level constraints:
  targetSeconds: number,                          // 15–60, creative-director picks per content type
  faceFirstHook: boolean,                         // first 3s must be face for stop-the-scroll
}

Validation lives in packages/shared/src/zod.ts as ReelShotlistSchema. creative-director output is parsed and rejected if non-conforming.

Pacing guidance (lives in the creative-director prompt, not the schema):

  • Hook / origin / confessional / bilingual punch: 15–25s
  • Data reveal / single-stat unpack: 25–35s
  • Framework explainer / Bug Series / MFR: 40–60s
  • Rule: one idea per Reel. If it can't be told in 60s, it's a Carousel, not a Reel.

5.2 reel-render-worker prompt (new)

Sibling to render-worker.md. Spawned by orchestrator P2-render fanout when unit.format === "Reel".

Worker input contract (staged via ctx.stageInput):

{
  runId: string,
  scriptId: string,
  variantId: string,                              // variantId(scriptId, "Reel", "9:16")
  format: "Reel",
  aspect: "9:16",
  width: 1080,
  height: 1920,
  language: "en",
  targetSeconds: number,
  faceFirstHook: boolean,
  scenes: Array<{                                // projected from shotlistEn
    scene: string,
    voiceover: string,
    onScreenText: string,
    chartRef: string | null,
    sceneType: "face" | "chart" | "face-over-chart",
    estimatedSeconds: number,
  }>,
  heygen: {
    avatarId: string,                             // env: HEYGEN_AVATAR_ID
    voiceId: string,                              // env: HEYGEN_VOICE_ID
  },
}

Worker output contract (folded into CreativeVariant row):

{
  variantId: string,
  scriptId: string,
  format: "Reel",
  aspect: "9:16",
  assetFiles: [{ url: string, kind: "video", duration: number }],
  durationSeconds: number,
  reelHeygenJobId: string,
  sceneCuts: Array<{ atSeconds: number, sceneIndex: number, type: "face" | "chart" }>,
}

5.3 packages/media-stitch (new)

Pure library. ffmpeg orchestration. No I/O outside its working dir + caller-supplied URLs. Sibling to packages/static-renderer.

export interface StitchSpec {
  workDir: string;                                // tmpdir for downloads + intermediate frames
  output: { width: number; height: number; durationSeconds: number };
  audioTrack: { url: string };                    // single HeyGen mp4 — audio extracted by ffmpeg
  cuts: Array<                                    // sorted by start
    | { type: "face",  startSec: number, endSec: number, source: { url: string } }
    | { type: "chart", startSec: number, endSec: number, framePngPath: string }
  >;
  subtitles?: { srtPath: string };                // burned-in EN subs
  music?: { url: string, gainDb: number };        // optional bg music
}

export interface StitchResult {
  mp4Path: string;                                // local file path; caller uploads via asset-store
  durationSeconds: number;
  warnings: string[];
}

export async function stitch(spec: StitchSpec): Promise<StitchResult>;

Boundary discipline. No HeyGen client logic. No asset-store calls. No store writes. Deterministic: same StitchSpec → same MP4 bytes (modulo ffmpeg version pinned via Docker image tag).

5.4 mcp-servers/heygen-wrapper (lightly extended)

Already generates video + polls status. Extension: getVideoStatus surfaces subtitleUrl + durationSeconds when HeyGen returns them.

{ status: "completed", videoUrl: string, subtitleUrl?: string, durationSeconds?: number }

If HeyGen does not return SRT for the account tier, the worker falls back to whisper-based force-alignment (see §6 Data flow).

5.5 Unchanged components

  • mcp-servers/static-renderer — called once per chartRef scene with { width: 1080, height: 1920, chartRef, language: "en" }; returns PNG path.
  • mcp-servers/asset-store — final MP4 upload returns CDN URL → assetFiles[0].url.

6. Data flow & sequencing

Phase 0 — Plan (creative-director, P1-fanout)

Input: Script, Brief, hook bank. Output: CreativeUnit validated against ReelShotlistSchema. Bad output rejects at the orchestrator boundary. No external calls yet.

Phase 1 — Orchestrator stages worker input (P2-render, in process)

const reels = plan.creatives.filter(c => c.format === "Reel");
// existing static path handles Feed + Carousel unchanged
const reelUnits = await Promise.all(reels.map(async (unit, i) => {
  const inputRef = await ctx.stageInput(`reel-${i}`, reelWorkerInput(unit, run.runId));
  return { spawnPrompt: reelSpawnPrompt(inputRef) };
}));

ADR-024 compliance: spawnPrompt carries only the sr_… ref, never the inlined spec.

Phase 2 — Reel worker executes (the meat)

One worker invocation. Steps run in sequence with explicit retries.

Step 1 — Hydrate input

mcp__orchestrator__read_step_result → ReelWorkerInput.

Step 2 — Compose HeyGen script

Concatenate scenes[].voiceover with sentence-boundary markers between scenes. Single HeyGen call → one continuous narration matching the chosen audio model.

Step 3 — Generate HeyGen video

mcp__heygen__generate_video({ avatar_id, voice_id, input_text: scriptText, language: "en", aspect_ratio: "9:16" }) → { jobId }.

Step 3a — Persist jobId BEFORE polling. Fix for the orphaned-Reels scar at exec.ts:92. After HeyGen returns { jobId }, immediately: 
mcp__store__update({
  entity: "CreativeVariants",
  id: variantId,
  props: { reelHeygenJobId: jobId, renderState: "HeygenGenerating", renderStartedAt: now() }
})
THEN begin polling. If the worker dies mid-poll, a future /produce --run=<id> pass resumes from the persisted jobId.

Poll mcp__heygen__get_video_status every 10s, max 5 min, until status: "completed" → { videoUrl, subtitleUrl?, durationSeconds? }.

Step 4 — Acquire word-level timings

  • If subtitleUrl present: fetch + parse SRT → word/phrase timing array.
  • Else: download videoUrl → audio extract → whisper transcribe with --word_timestamps → word timing array.

Result: WordTimings = Array<{ word, startSec, endSec }>.

Step 5 — Map scenes to time ranges

For each scene, string-match the first/last words of scene.voiceover against the WordTimings stream to derive startSec / endSec. If match fails for any scene, fall back to proportional allocation (sceneIndex / sceneCount × duration) and add a warning.

Correctness rule: the creative-director's estimatedSeconds is a planning hint, NOT a source of truth for cut placement. Real timing comes from word-aligned text.

Step 6 — Render chart frames

For each scene where sceneType ∈ {chart, face-over-chart} AND chartRef !== null: one mcp__static-renderer__render_html_to_png call returning a framePngPath. Reuses existing chart YAMLs in corpus/data/charts/.

Step 7 — Build StitchSpec & stitch

cuts = sceneTimeline.map(s => {
  if (s.sceneType === "face") return {
    type: "face", startSec, endSec, source: { url: videoUrl }
  };
  if (s.sceneType === "chart") return {
    type: "chart", startSec, endSec, framePngPath
  };
  // face-over-chart: emit two layered cuts (PiP via overlay filter)
});

stitch({
  workDir: /tmp/run_{runId}/reel_{variantId},
  output: { width: 1080, height: 1920, durationSeconds: total },
  audioTrack: { url: videoUrl },                  // audio extracted from HeyGen mp4
  cuts,
  subtitles: { srtPath: writeSrtFromWordTimings(...) },
}) → { mp4Path, durationSeconds, warnings[] }
Correctness rule: HeyGen's mp4 carries both the face video AND the narration audio. Stitcher extracts audio from that one file (ffmpeg -map 0:a) and overlays new video frames during chart cuts. Never re-synthesize audio — that drifts lip-sync.

Step 8 — Upload & return

mcp__asset-store__upload({ filePath: mp4Path, kind: "video" }) → { url: cdnUrl }. Return JSON with assetFiles: [{ url: cdnUrl, kind: "video", duration }].

Phase 3 — Orchestrator writes CreativeVariant (P3-write, existing)

Existing P3-write step folds worker JSON into CreativeVariant rows. assetFiles is already a field; the orchestrator doesn't care that it's a video.

Phase 4 — HG3 gate (existing)

Webapp variant review page renders <video src={assetFiles[0].url} controls />. Approve → schedule + distribute downstream.

Phase 5 — Distribute (unchanged, already on main)

Reel assetFiles[0].url flows into mcp__meta-ads__upload_video, mcp__meta-organic__publish_video_post, and the new Distributions audit log. Zero changes required in this spec.

Orphan recovery — explicit fix for exec.ts:92

Two new columns on CreativeVariants:

  • reelHeygenJobId: text nullable
  • renderState: text nullable; enum: null | "HeygenGenerating" | "HeygenCompleted" | "Stitching" | "Uploaded" | "RenderFailed"

(null means static-renderer path or not started — backward-compatible with existing Feed/Carousel rows.)

Worker writes reelHeygenJobId immediately on Step 3a. If the worker dies mid-poll, a new /produce --run=<id> pass detects rows in renderState="HeygenGenerating" and resumes polling from the persisted jobId. The HeyGen result is never lost.

Idempotency

StepIdempotent?How
HeyGen generateWithin a runCheck row reelHeygenJobId; if set, skip submit and poll instead
Chart frame renderNaturallystatic-renderer is pure; same chartRef + dims = same PNG
StitchDeterministicSame StitchSpec = same MP4 bytes (modulo ffmpeg version)
Asset-store uploadContent-hashedDuplicate uploads return existing URL

Re-running /produce --run=<id> on a partially-failed run is safe.

Things that intentionally do NOT auto-retry

  • HeyGen status: failed — usually script content tripped moderation; auto-retry burns money. Surface to HG3 reviewer.
  • ffmpeg failure — deterministic; re-running reproduces.
  • Whisper missing — system-config issue, not per-Reel.

8. Testing strategy

Layer 1 — Pure libraries (vitest, no I/O)

  • ReelShotlistSchema: accept valid, reject targetSeconds out of 15–60, reject chart scenes with null chartRef, reject voiceover > 30 words.
  • build-filtergraph.ts: StitchSpec → ffmpeg argv. Snapshot tests on argv for each cut combination (face only, mixed, face-over-chart PiP, with subtitles, with music).
  • scene-to-time.ts: exact match, punctuation-fuzzy match, partial-match-falls-to-proportional, empty-transcript edge case.

Layer 2 — media-stitch integration (vitest, Docker required)

One e2e test that invokes ffmpeg in Docker against tiny fixtures: 3s test mp4 (HeyGen-shaped), 1 chart PNG, 5-line SRT. Asserts ffprobe reports correct resolution + codec; subtitle track present. Marked integration — skipped when DOCKER_HOST unavailable.

Layer 3 — Orchestrator stage tests (produce.test.ts extension)

  • Mixed plan: 2 Scripts × 4 CreativeUnits = 8 units, 2 of which are Reels — asserts P2-render emits correct static vs reel spawns.
  • ADR-024 compliance: spawn prompts carry only sr_… refs.
  • Orphan resume: pre-existing renderState="HeygenGenerating" + jobId → emits resume spawn, not fresh.

Layer 4 — Worker fixture tests

Five fixtures under test/fixtures/reel/:

  • reel-pure-face.fixture.ts — 0 chart scenes
  • reel-mixed.fixture.ts — 2 face + 2 chart scenes
  • reel-pic-in-pic.fixture.ts — face-over-chart PiP
  • reel-resume.fixture.ts — pre-existing reelHeygenJobId; expects skip-to-poll path
  • reel-heygen-failed.fixture.ts — HeyGen mock returns failed; expects RenderFailed row state

MCPs mocked except media-stitch::stitch — runs against real fixture mp4 in Docker (exercises Layer 2).

Layer 5 — Schema migration

Up: existing rows get null for new columns. Down: clean drop. pnpm lint:migrations passes.

Layer 6 — End-to-end smoke (manual, gated)

One real HeyGen call before merging PR 4: pnpm tsx scripts/smoke-reel.mts --script="..." --avatar=HEYGEN_AVATAR_ID_TEST. Not in CI (costs money).
