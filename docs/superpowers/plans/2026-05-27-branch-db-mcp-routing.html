2 Implement db-guard helper

Files

  • packages/shared/src/db-guard.ts (new)
  • packages/shared/src/index.ts — re-export

What

export interface GuardOpts {
  branch?: string | null;     // explicit override (tests)
  allowLive?: boolean;        // explicit override (tests)
  cwd?: string;               // for .git/HEAD walk
}

export function assertDbSafeForBranch(
  databaseUrl: string,
  opts: GuardOpts = {},
): void {
  const allowLive = opts.allowLive ?? process.env.ALLOW_LIVE_DB === "1";
  if (allowLive) return;

  const dbName = parseDbName(databaseUrl);
  if (dbName !== "engineerdad") return; // any suffixed DB is safe

  const branch = opts.branch ?? detectBranch(opts.cwd ?? process.cwd());
  if (branch === null) return;          // CI / production / no .git
  if (branch === "main") return;

  throw new Error(
    `Refusing to connect to live engineerdad DB from branch ${branch}. ` +
    `Run: pnpm db:sandbox && pnpm db:snapshot, or set ALLOW_LIVE_DB=1.`,
  );
}

parseDbName uses new URL(databaseUrl) and trims the leading / from pathname. detectBranch walks up from cwd looking for .git/HEAD; parses ref: refs/heads/<name>; returns null if missing or detached.

What

In each getDb() (or the equivalent lazy constructor), before calling postgres(DATABASE_URL, …), call assertDbSafeForBranch(DATABASE_URL). 5 Snapshot script (db:snapshot)

Files

  • scripts/db-snapshot.mjs (new)
  • package.json (root) — add script entry

What

ESM script. Logic:

  1. import { branchSlug } from "./lib/db-slug.mjs"; resolve target.
  2. Read the current branch from .git/HEAD; if main, exit with message “nothing to snapshot to”.
  3. Verify the target DB exists by connecting briefly (using postgres client) and querying pg_database. If missing, exit non-zero with “run pnpm db:sandbox first”.
  4. Define SRC=engineerdad, TGT=<slug>, TMP=/tmp/<slug>.dump.
  5. execSync the dump: 
    docker exec engineerdad-postgres bash -c \
  "pg_dump --no-owner --no-acl --clean --if-exists -Fc \
     -U engineerdad -d ${SRC} -f ${TMP}"
  6. execSync the restore: 
    docker exec engineerdad-postgres bash -c \
  "pg_restore --no-owner --no-acl --clean --if-exists \
     -U engineerdad -d ${TGT} ${TMP}"
  7. Re-apply the same Drizzle pushes that db-sandbox.mjs does (the schema-push helper can be refactored out or duplicated — duplication is fine; ~3 lines).
  8. docker exec engineerdad-postgres rm ${TMP}.
  9. Print “Snapshotted engineerdad → <slug>” on success.

Set process.env.ALLOW_LIVE_DB = "1" at the top of the script if it ever opens a JS-side connection to engineerdad (the existence-check uses the admin postgres DB, not engineerdad, so likely not needed — but cheap insurance).

package.json root, scripts section:

"db:snapshot": "node scripts/db-snapshot.mjs"

Acceptance

Done when: pnpm db:snapshot on this branch completes without error, and docker exec engineerdad-postgres psql -U engineerdad -d <slug> -c "SELECT count(*) FROM orchestrator.runs" shows the same count as live.

What

In the “Database workflow” section, add three bullets and lightly edit:

For the live DB use -d engineerdad; for the current branch sandbox use -d $(node -e 'import("./scripts/lib/db-slug.mjs").then(m => console.log(m.branchSlug()))') or just paste the URL from .env.local.

3. Rollback

If something goes wrong post-merge:

  1. Revert the commit. .mcp.json reverts to live-only; guard helper goes away.
  2. No data migration needed — branch sandboxes are throwaway; live engineerdad was never modified by the rollback path.
  3. Restart Claude Code.
