Roadmap split — confirmed 2026-05-22 The original Phase 5 silently bundled two bodies of work. It is now split: Phase 5 (this doc) decomposes media-production into a taste-only agent + pure spec code + render workers — the produce pipeline. Phase 6 builds the three missing agentic-cell stages (synthesize, brief, content), assembles the ordered live registry, upgrades HG4, and deletes the replaced agents. Phase 7 trims brain.md + writes ADR-020. The roadmap moves 6 → 7 phases.
Three decisions baked in 1 — the taste/code line. creative-director (agent) owns only judgement: storyboard decomposition, hook rotation across the 4 distinct creatives, thumbnail briefs, chartRef reasoning, palette emphasis. Everything mechanical — the 5-row format matrix, Channels, the deterministic variantId, Meta/YouTube/organic copy specs, cost roll-up — is pure code in deriveSpecs. Litmus: if two careful people would write the same value, it is code.
2 — most atomics already exist. Phase 1 already shipped deriveChannels, metaCtaType, youtubeCategoryId, variantId, slugify, readingTime, topicTag, extractKeywords, truncateAtWord. deriveSpecs and deriveArticlePackaging compose them — they are thin.
3 — the contract becomes a unit test. The 706-line agent's F2 / G10 / G11 / Step-7.0 self-policing prose is superseded by verifyProduce — reading Notion + the events table for ground truth. C1 holds: the produce stage is wired to its real agent here, but the live registry stays fixture-only and media-production.md is not deleted until the Phase 6 cutover — deleting it now would dangle brain.md §E.

Goal: Replace the 706-line media-production.md with a ~80-line taste-only creative-director agent, a pure deriveSpecs / deriveArticlePackaging spec layer, render-worker prompts, and a verifyProduce that reads ground truth — so the produce stage runs end-to-end against real code.

Architecture: The produce stage keeps its Phase 2 shape (P1 spawn → P2 fanout → P3 write → P4 gate). Phase 5 makes each step real: P1 spawns the new creative-director, P2 fans out render-worker units, P3 runs deriveSpecs + the article pass, the verifier reads Notion. No engine change.

Tech stack: TypeScript (ESM), vitest. New deps: none. Execution: TBD — this doc is for review first.

1. Locked interfaces

1.1 — CreativePlan — the creative-director's return shape

One CreativeUnit per distinct creative (4 per Script — the Carousel pair is one creative). It carries the taste output and the source Script/Brief fields deriveSpecs needs, so deriveSpecs takes only the plan + render results.

export interface SceneCard {
  scene: number; durationSec: number; visual: string;
  onScreenText: string; voiceover: string; shotNotes: string;
  chartRef: string | null;
}
export interface CreativeUnit {
  scriptId: string;
  format: "Reel" | "Feed" | "YT-Long" | "Carousel";
  hook: { en: string; ms: string; register: string };
  shotlistEn: SceneCard[];          // 3–8 cards
  shotlistBm: SceneCard[];          // parallel, same scene count
  thumbnailBrief: string;
  paletteEmphasis: string;
  estCostMyr: number;               // 4c midpoint table
  source: {                         // copied from Script + parent Brief
    scriptBodyEn: string; scriptBodyMs: string;
    ctaEn: string; ctaMs: string;
    funnelStage: "TOFU" | "MOFU" | "BOFU";
    persona: string; topic: string; targetQuery: string;
    primaryLang: "en" | "ms";
  };
}
export interface CreativePlan { runId: string; creatives: CreativeUnit[]; }

1.2 — VariantSpec + deriveSpecs (derive/specs.ts)

export interface MetaCopySpec {
  primaryTextEn: string; primaryTextMs: string;   // ≤180, regulator phrase baked in
  headlineEn: string; headlineMs: string;         // ≤40
  descriptionEn: string; descriptionMs: string;   // ≤30
  ctaType: string;                                // metaCtaType()
  targetingJson: string;                          // persona → broad-targeting JSON
}
export interface YtCopySpec {
  title: string; description: string; tags: string[]; category: string;
}
export interface OrganicCopySpec {
  language: "EN" | "BM";
  captionEn: string; captionMs: string;
  hashtagsIg: string[]; hashtagsFb: string[];
}
export interface VariantSpec {
  variantId: string;                 // variantId(scriptId, format, aspect)
  scriptId: string; format: string; aspect: string;
  channels: string[];                // deriveChannels()
  estCostMyr: number;
  shotlistEn: SceneCard[]; shotlistBm: SceneCard[]; thumbnailBrief: string;
  assetFiles: { url: string; sha256: string }[];   // joined from render results
  meta: MetaCopySpec | null;         // when channels ∋ Meta-*
  yt: YtCopySpec | null;             // when channels ∋ YouTube*
  organic: OrganicCopySpec | null;   // when channels ∋ Meta-organic
}

export interface RenderResult { variantId: string; url: string; sha256: string; }

/** Expand each CreativeUnit into the 5-row format matrix (Carousel → 4:5 + 1:1),
 *  derive Channels + per-channel copy, join asset files. Pure. */
export function deriveSpecs(plan: CreativePlan, renders: RenderResult[]): VariantSpec[]

The 5-row matrix (Reel 9:16 · Feed 4:5 · YT-Long 16:9 · Carousel 4:5 · Carousel 1:1) is the fixed expansion; the Carousel pair shares one CreativeUnit's shotlist. Each branch is keyed off Channels — exactly the media-production §5.7/§5.8 conditionals, now in code.
