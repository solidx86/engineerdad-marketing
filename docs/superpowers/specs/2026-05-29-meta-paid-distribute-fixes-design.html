Context

Doctrine grounding (ADR-022 / 023 / 024)

The unified rule: the conductor is a reasoning surface; bulk and integration detail never cross its tool channel; deploy-config lives in the execution layer (packages/<name>), reached by in-process calls — not pushed through step_results or the conductor.

For Defect A this decides where page_id/link_url are injected:

Chosen — MCP defaults from env. writes.ts already reads AD_ACCOUNT_ID and META_TOKEN via requireEnv() at this exact layer. page_id/link_url are the same class of Meta deploy-config. Injecting them here colocates Meta config in one place (ADR-005 thin-adapter, ADR-023 "integration detail lives in the package"), and the values never enter step args, never get staged to step_results, never touch the conductor — so ADR-022/024 are not engaged at all.
Rejected — planner bakes from env. creativeStep() runs inside build(); ADR-024 enumerates what build() may do (pure computation, ctx.stageInput) and reading process.env is not among them. The baked values would then flow into the ADR-024 staged-input payload → persisted in step_results → read back by the worker, using the claim-check substrate as a config carrier (out of ADR-022's stated scope) and splitting Meta config across two layers.

The flip case — a per-campaign landing URL — would be ADR-018 routing territory and belong in the planner or a Brief field. We chose a static LANDING_URL, so it is deploy-config and lands in the MCP layer.

The fixes

Fix A — create_ad_creative deploy-config Option 1

  1. mcp-servers/meta-ads/src/index.ts — make page_id and link_url schema fields optional (z.string().min(1).optional() / z.string().url().optional()). Keep call_to_action accepting both the object shape and a bare string (see step 2).
  2. packages/meta-ads/src/writes.ts createAdCreative — when page_id is absent, default to requireEnv("META_ORGANIC_PAGE_ID"); when link_url is absent, default to requireEnv("LANDING_URL"). Normalize call_to_action: if it arrives as a string ctaType, wrap to { type: ctaType, value: { link: <resolved link_url> } }; if it arrives as the object shape, pass through (filling value.link from the resolved link_url when missing).
  3. plan-distribution.ts creativeStep() — leave page_id/link_url out (the MCP fills them); keep call_to_action: spec.ctaType as the bare string (MCP normalizes).
  4. .env — add LANDING_URL=https://engineerdad.my.

Fix B — adset bid_strategy + Advantage-Audience

  1. plan-distribution.ts adsetStep() — add bid_strategy: "LOWEST_COST_WITHOUT_CAP" (the no-bid-amount default; safe with daily_budget_cents and no bid_amount).
  2. plan-distribution.ts targetingForCell() — add targeting_automation: { advantage_audience: 0 } to the returned targeting object, preserving the explicit age/locale/geo targeting (no Meta auto-expansion).
  3. Confirm createAdset (writes.ts) forwards bid_strategy (it already accepts the field) and passes targeting through verbatim (so targeting_automation reaches the API).

These are deterministic paid-campaign policy with no env dependency, so they belong in the planner (ADR-018: the planner owns mechanical routing/policy; the MCP executes).

Budget model — deliberately per-adset (ABO), not campaign-level (CBO). The daily budget is set on each adset via the existing daily_budget_cents in adsetStep() — there is one adset per experiment cell (angle). On the walk this materialized as RM51.86 × 3 cells + RM44.44 = RM200.02/day total, where dailyBudgetMyr = 200 comes from Brain's experimentParams.dailyBudgetMyr (Fix B does not touch this — it only adds bid_strategy + the Advantage-Audience flag). The campaign carries no budget: CBO is off (is_adset_budget_sharing_enabled defaults false, per the B-025/26/27 work), and ads inherit their adset's budget — budget is never set per-ad.

This is intentional for a fair angle experiment: per-adset budgeting guarantees each angle-cell gets a fixed, controlled daily spend, so the experiment's per-angle signal isn't confounded by Meta reallocating a shared pool toward an early front-runner. CBO (one campaign-level RM200/day pool that Meta optimizes across angles) would maximize aggregate results but bias spend toward whichever angle Meta favours early — defeating the per-angle readout the experiment exists to produce. We therefore keep ABO.

Note on bid_strategy: "LOWEST_COST_WITHOUT_CAP": "WITHOUT_CAP" means no per-result bid cap, not "no budget". The per-adset daily_budget_cents is a hard spend ceiling that this bid strategy operates within. The bid-capped alternatives (COST_CAP / LOWEST_COST_WITH_BID_CAP) require a bid_amount — the field whose absence made Meta reject the adsets on the walk — so the no-bid-cap default is the correct choice here. (ADR-015 also keeps every entity PAUSED, so $0 actually spends until a human activates in Ads Manager.)

Fix C — D2a/D2b worker contract (ADR-022 compliance)

  1. distribute.ts setupPromptFor() — append the ADR-022 contract: "Before your final message, call mcp__orchestrator__write_step_result({ runId, stepId, payload }) with your result object as payload, then return only { stepResultId }." Mirror the wording in corpus/templates/worker-prompts/render-worker.md.
  2. D2b route prompt (the routePromptFor-equivalent in distribute.ts) — same contract appended. The D2b workers return { rowId, channel, status, … } inline today; they must persist + return a ref.
  3. This removes the conductor-side write_step_result workaround used during the walk. The verify/advance path then receives proper refs for both steps.

Fix D — organic caption missing compliance footer (prompt rule + produce verifier)

Investigation result: there is no code-clipping bug. produce.ts:392 stores organicCaptionEn = v.organic.captionEn verbatim from the creative-director's CreativePlan — no slice/substring/column limit anywhere in the produce path. The truncated caption ("…does not constitute personal financial") was emitted that way by the creative-director agent: it generated a long caption that ran out before the compliance footer. Root cause is authoring, and the failure only surfaced at the Meta-organic publish boundary (late, after the whole pipeline ran).

Two-layer fix, matching the repo's compliance defense-in-depth (agent prompt + verifier, as Fix B-016 did for briefs):

  1. Prompt rule — .claude/agents/creative-director.md (and its prompt fragment under packages/shared/src/prompts/ if sourced there; run pnpm sync:agents after): a HARD RULE that every organic caption (captionEn / captionMs) MUST end with the full bilingual FIMM/SC compliance footer — consultant credential (Shoo Kyuk Wei + Public Mutual + FIMM/UTC), the past-performance risk warning, and the prospectus/PHS pointer — and must never be truncated.
  2. Produce-stage verifier (deterministic backstop) — extend verifyProduce() (packages/orchestrator/src/verifiers/verify-produce.ts) to run checkCompliance() from @engineerdad/meta-ads (already an orchestrator dependency — package.json:24) on the organicCaptionEn / organicCaptionBm of every organic-channel variant. A caption missing its footer contains none of the sentinel regulator phrases (past performance, fimm, public mutual, …) and fails — surfacing the problem at HG3, in-pipeline, instead of at the Meta boundary. This requires carrying the caption text into the P5-confirm projection (today projectVariant reduces it to a boolean organicSpecComplete; add the raw organicCaptionEn/Bm fields and the EN/MS channel flag the check needs).

Layering: the produce verifier is the early sentinel check (catches truncation/footer-missing); the meta-organic publish gate stays as the authoritative final check (the strict 3-block consultant/risk/prospectus preflight) — unchanged. The fix is upstream of that gate, not in it.

Testing strategy

TDD per fix (red → green):

  • Fix A — unit tests on createAdCreative: (i) absent page_id/link_url default from env; (ii) bare-string call_to_action normalizes to the object shape with value.link; (iii) object-shape CTA passes through. Update the planMetaPaid integration test to assert creativeStep() no longer needs the fields and the routed plan is well-formed.
  • Fix B — unit tests on adsetStep() (asserts bid_strategy) and targetingForCell() (asserts targeting_automation.advantage_audience: 0 alongside the existing geo/age/locales). Extend the planMetaPaid integration test.
  • Fix C — assert setupPromptFor() and the D2b route prompt contain the write_step_result + {stepResultId} contract line (string-contains test, consistent with how render-worker prompt expectations are tested).
  • Fix D — (i) verify-produce unit test: a variant whose organicCaptionEn is truncated/footer-less fails verification; one with the full footer passes; non-organic-channel variants are unaffected. (ii) produce.ts projectVariant test asserting the raw caption fields are carried through P5-confirm. (iii) creative-director prompt assertion (string-contains for the new HARD RULE) consistent with how brief-writer's canonical-angle rule is tested.
Out-of-scope acceptance: the Meta app capability gap is external. The plan cannot green a live Meta write. Final acceptance is re-running the failed D2b rows on run_1780029260 (idempotent — all rows are adId/fbPostId null, retry-safe) once ads_management standard access is granted.

Rollout
