Bilingual policy (ADR-010)

  • EN + BM — items 1, 2, 5 (consumer-facing methodology and proof)
  • EN only — items 3, 4, 6, 8, 9, 10 (author-facing templates, aggregate snapshots, structured data)
  • EN only — items A–G (pure data files, no consumer-facing prose)

Naming

  • Date suffix only on corpus/data/funds-apr2026.json; other dated snapshots carry the date in frontmatter so the refresh script overwrites cleanly

Indexing & retrieval

After every write to /corpus, run /ingest-corpus to regenerate corpus/.index/chunks.jsonl and the BM25 sidecar. The chunker is language-aware — ## English and ## Bahasa Malaysia sections become separate chunks tagged by lang.

At runtime, agents call mcp__corpus__search({ query, scope, lang, k }) — a BM25 lookup that returns only the top-k chunks relevant to the query (content-gen uses k: 12, per content-gen.md). Agents never read whole .md files; per-call context is bounded at roughly 12 × 800 tokens ≈ 10k tokens of grounding regardless of corpus size. Adding files grows the searchable universe, not the per-call context.

Indexer file types — the chunker only ingests .md, .pdf, .txt, .vtt. Items 10 and A–G (everything under corpus/data/) are not BM25-indexed; consumers read them directly via the Read tool when they need specific facts. The aggregate-prose layer (items 6, 8) is indexed and surfaces those facts via search.

Per-artifact spec

1. EPF sustainability model (methodology)

🟢 compliance-safe
corpus/courses/epf-sustainability-model.md Language EN + BM Dated No

Purpose — Documents the math behind the EPF Sustainability Calculator. Pairs with proof/epf-baseline-tiers.md (which carries the numerical tiers; this carries the engine that produces them).

Source — epf-sustainability-calculator/index.html, constants block (lines 787–798) and the simulator function.

Sections (EN + BM mirror)

  • Contribution math (employer 13% if salary < RM5,000, else 12%; employee 11%)
  • Accumulation phase: monthly contributions × historical EPF dividend (assumption: 6% long-run nominal)
  • Drawdown phase: retirement spend = Belanjawanku Adequate tier inflated forward at 3%/yr
  • Sustainability test: balance after each retirement year must remain ≥ 0 until life-expectancy assumption
  • Investment alternative: 8% nominal long-run for unit-trust contribution math, with explicit volatility caveat
  • Assumption block (one canonical place — referenced by item 2): EPF dividend 6%, salary growth 3%/yr, inflation 3%/yr, retirement age 60, life expectancy 80
Compliance discipline — frame all rates as modelling assumptions, never as guaranteed returns. Never claim PRS or unit-trust returns are "safer" or "beat" EPF. Cite Belanjawanku / KWSP as the source of the spend tier.

5. Weighted-alpha scoring (methodology)

🟢 compliance-safe
corpus/courses/weighted-alpha-scoring.md Language EN + BM Dated No

Purpose — Documents the screening rubric used in the Fund Master. Companions mfr-framework.md (which is conceptual; this is mechanical).

Sections (EN + BM mirror)

  • The weighting: YTD 5% · 1Y 15% · 3Y 40% · 5Y 25% · 10Y 15% — and why 3Y dominates (full cycle without overfit to ancient data)

Data layer — items A–G

Seven additional structured-data files under corpus/data/. They prevent content-gen from fabricating arithmetic by giving it pre-computed lookup tables and verified reference values. Like item 10, none of these are BM25-indexed — consumers reach them via the Read tool.

A. Compounding table

🟢 evergreen · script-generated
corpus/data/compounding-table.json Language EN Dated No

Purpose — Pre-computed future-value-of-annuity table. Prevents fabrication of "RM X/mo at 8% over 20 yrs = ?" claims that show up in education-fund and retirement-shortfall posts.

Grid: annual rate ∈ {6%, 7%, 8%, 10%} × years ∈ {5, 10, 15, 20, 25, 30, 35} × monthly contribution ∈ {RM100, 200, 300, 500, 750, 1000, 1500, 2000}.

Generator: scripts/build-derivative-tables.mjs. Source assumptions: corpus/courses/epf-sustainability-model.md (item 1).

Build & refresh scripts

Three scripts now sit at scripts/, each with a distinct cadence:

1. scripts/refresh-funds-corpus.mjs — monthly

Produces: items 6, 8, 10. Cadence: monthly when a new MFR xlsx drops.

  1. Locate the most recent PublicMutual_FundMaster_*.xlsx in ~/Downloads/Funds/output/fundmasters/ by mtime
  2. Parse Master sheet → in-memory fund records
  3. Write corpus/data/funds-<period>.json (period derived from filename)
  4. Write corpus/proof/fund-universe-stats-snapshot.md + corpus/proof/sector-geo-rotation-snapshot.md; include a vs_prior diff if a prior snapshot exists in git history
  5. Print summary; exit 0
  6. Does not run /ingest-corpus — deliberate human step

Dependency: exceljs. Invoked via: pnpm refresh:funds.

2. scripts/harvest-epf-simulations.mjs — when calculator version changes

Produces: item G — corpus/data/epf-sustainability-simulations.json. Cadence: rare (only when APP_VERSION in the calculator changes).

  1. Spawn a local HTTP server in engineerdad-site/tools/epf-sustainability-calculator/ (e.g. python3 -m http.server on a free port via child_process)
  2. Drive headless Chromium via the Playwright MCP tools to open the calculator
  3. Preferred path — browser_evaluate calls the calculator's internal simulator function directly (e.g. window.simulator(inputs)) for each grid tuple; pure JS calls, ~milliseconds each
  4. Fallback path — browser_fill_form + click "Calculate" + read DOM output if the calculator doesn't expose a JS-callable entry point
  5. Iterate the grid (~1,500 tuples): age × salary × current-balance bucket × retirement age × lifestyle tier
  6. Write JSON including calculator_version (from APP_VERSION) and harvested_at ISO timestamp
  7. Tear down the HTTP server; exit 0

Boundary discipline — driving the calculator via Playwright, not re-implementing its math, keeps us on the right side of ADR-016 (we don't import source from a sibling repo; we consume it as a runtime artifact). The JSON is a cached snapshot of the calculator's output for a fixed grid.

Invoked via: pnpm harvest:epf.

3. scripts/build-derivative-tables.mjs — when item 1's assumption block changes

Produces: items A (compounding-table.json) and B (monthly-contribution-required.json). Cadence: rare (only when canonical assumption block in item 1 changes).

  1. Hardcode the assumption set with a comment pointing to corpus/courses/epf-sustainability-model.md as the canonical source
  2. Compute FV-of-annuity grid at 6% / 7% / 8% / 10% × 5/10/15/20/25/30/35 yrs × monthly contribution rates → item A
  3. Compute inverse (required monthly pmt to reach FV) over the same grid → item B
  4. Write both JSONs with assumption_source: "corpus/courses/epf-sustainability-model.md"

Invoked via: pnpm build:tables.

Acceptance criteria

Prose layer (items 1–6, 8, 9, 10)

  • Nine artifacts exist at the paths listed (item 7 deferred)
  • All .md files carry conformant frontmatter; dated ones carry dated_snapshot: true + snapshot_period
  • Items 1, 2, 5 have working bilingual EN + BM sections
  • No file recommends a specific fund or compares competitor funds
  • No file presents projected returns as guarantees
  • Every numerical claim in item 2 is traceable to a row in item G, item C, item B, or item D — no in-prose arithmetic
  • Item 9 omits the anomaly section (with a note pointing to its future addition); other sections present and sourced

Data layer (items A–G)

  • Seven data files exist under corpus/data/
  • Items A + B generated deterministically by scripts/build-derivative-tables.mjs
  • Item G generated by scripts/harvest-epf-simulations.mjs — Playwright-driven against the live calculator, JSON carries calculator_version matching the calculator's APP_VERSION
  • Item G parity spot-check — at least 3 random tuples produce outputs that match the live calculator UI when input manually
  • Items C, D, E, F carry verified_at + source fields; F also carries year
  • None of A–G are BM25-indexed (corpus chunker doesn't ingest .json)

Scripts & infra

  • scripts/refresh-funds-corpus.mjs runs end-to-end against the Apr 2026 xlsx and produces items 6, 8, 10 deterministically
  • scripts/build-derivative-tables.mjs runs and produces items A, B
  • scripts/harvest-epf-simulations.mjs runs and produces item G
  • pnpm test passes
  • /ingest-corpus runs cleanly after the write phase; BM25 index grows by at minimum the chunks-worth of items 1, 2, 5, 6, 8 (the BM25-indexed additions in this iteration)
  • Nothing pushed without separate approval

Out of scope

  • Item 7 — fund anomaly cases (deferred to a later iteration)
  • A web UI for the corpus refresh
  • MCP tools wrapping any of the three scripts (they're pnpm tasks, not MCP servers)
  • Migrating other xlsx artifacts in /Users/solid/Downloads/Funds/ (only the most-recent output/fundmasters/*.xlsx is read)
  • Per-fund recommendation files (FIMM / SC-banned)
  • Backfilling Feb 2026 and Mar 2026 snapshots — the script can do it on demand later, but the design ships with Apr 2026 only
  • Backfilling historical years for items E (dividend history) and F (tax reliefs) beyond a sensible 10-year seed
