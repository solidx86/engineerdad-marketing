4 · The shared partial — corpus/templates/worker-prompts/_chart-rules.md (new)

Format-agnostic chart doctrine, lifted out of both prompts (currently reel §95–106 / static §4 step + §5):

  • HTML legend block above the canvas; built-in Chart.js legend is OFF (legend.display=false). Never the in-canvas legend.
  • Line charts → line-style legend swatches (thin rounded bars whose heights mirror the line weights, not squares) + hero-line point markers (white-ringed); volatile comparison lines stay marker-less. (The rule shipped for reel-data-line in b3f63fa, now promoted to shared doctrine.)
  • Callout/badge: rendered verbatim from the YAML callout_en (figure OR framing line), as its own HTML row outside the plot rectangle — never floated over canvas/axis/curve. No callout_en → no badge.
  • Numbers + source_citation come verbatim from the YAML — never invented or recomputed.
  • buildChartConfig(yaml, tokens, {lang}) from corpus/templates/partials/chartjs-config.js; window.__chartsReady signal. Pie/donut hand-authored in the same spirit (not yet in buildChartConfig).
  • Tables / stat-grids: pure HTML, no Chart.js, contrast carried by the layout; the chart rules above do not apply.
Stays prompt-specific (not in the partial): caption rendering (static: render caption_en below chart / reel: never), aspect & scene model, carousel multi-card furniture, the safe-area constraint (reel-only).

Both prompts get a one-line pointer: "If any scene has a chartRef, also Read corpus/templates/worker-prompts/_chart-rules.md before composing."
