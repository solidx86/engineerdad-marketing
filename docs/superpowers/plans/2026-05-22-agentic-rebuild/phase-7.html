1. ADR-020 — what it records

ADR-020 is the durable artifact of this rebuild. Sourced from the two reviewed specs, it records what is now true in the code:

  • Context — the pre-rebuild model: brain as a 4-tier router dispatching 8 agents; deterministic logic living in agent prompts; the recurring G-series regressions; PLAN.md as a 754-line cited-by-section spec.
  • Decision — the orchestrator state machine (plan/verify/advance, stage definitions, verifiers) owns sequencing; four agentic cells (Strategy/brain, brief-writer, content-writer, creative-director) do the open-ended reasoning; everything else is a deterministic StageDefinition or a fan-out worker. The agentic litmus test: a step is agentic only if it has genuine open decision space — otherwise it is code.
  • Retained from ADR-005 — the thin-adapter doctrine, the no-MCP-mesh rule (restated: "no MCP server opens a client to another MCP server" — shared logic goes in packages/, cross-server sequencing through the hub), and the compliance boundary (the scan fires at the notion MCP write boundary).
  • Resolves — ADR-014's worker-split question (the ADR-014 + ADR-016 render-vs-handoff carve-out stands).
  • Consequences — /loop is the conductor; the granular commands are scoped delegators; the four human gates live inside stages as gate steps; orchestration state is SQLite-owned, Notion owns artifacts.
  • Status — Accepted, 2026-05-22. ADR-005 gets a one-line note: "Status note (2026-05-22): the dispatch model is superseded by ADR-020; the thin-adapter + no-MCP-mesh + compliance-boundary doctrine below stands."

2. Command surface — the rewrite

Phase 6 deleted brain.md §A (mode dispatch); every granular command that spawned brain with a mode: string is non-functional. Phase 7 rebuilds them in two shapes:

Command(s)New shape
/loop-once · /briefConductor delegator (start). Mint a fresh run and drive it via loop.md's procedure to HUMAN GATE 1.
/content · /produce · /experiment · /distributeConductor delegator (resume). Parse runId from $ARGUMENTS (--run=<id>); drive the run via loop.md's procedure to the next human gate. The body names the stage the command is the escape hatch for.
/analyze · /reflectScoped brain spawn. Off-loop reasoning ops. /reflect → spawn brain to run the §B-step-2 Reflect procedure on the given run. /analyze → spawn brain to summarise the analytics signal; experiment readout via mcp__experiment__readout when --run is given.
/brainKept. Generic brain spawn — still valid (brain is a reasoner). Description refreshed to drop the router framing.
/loop · /status · /ingest-corpus · /posting-packUnchanged. Already correct — conductor, run dashboard, corpus reindex, and the IG manual-posting aid (a script runner, no brain dispatch).
/post-week · /targetingDeleted. /post-week — the organic-planner flow dissolved by design §7. /targeting — a legacy-named duplicate of /brief.

The delegators carry allowed-tools: Task, mcp__orchestrator__* to match loop.md; they reference loop.md's loop body rather than restating it — one conductor, several front doors.
