Authoring Doctrine — applies to every entry

Hard rules. A reviewer rejects any entry that breaks them.

  1. EN ships first. lang_status: en_only on first commit. BM is a later pass, not in this plan.
  2. Proof discipline. Every entry carries at least one of: a real PMB factsheet / fund-code worked example, a prospectus or trust-deed clause citation, or an LHDN / FIMM / SC reference. No generic "unit trusts in general…" hedging.
  3. Honesty on figures. For source_type: public, author with best-known public PMB/LHDN figures and name the document in source_ref. Do not invent precise numbers as if confirmed — where an exact figure is used (a sales-charge %, a switching fee, a relief cap) it is flagged for Shoo to confirm in PR review. The sourcing blockquote says so explicitly.
  4. Synthesized framing. source_type: synthesized entries are plausible patterns, never real client cases. Body uses "imagine / a parent who / scenario" language. Never "my client."
  5. Hook density. Every granularity: concept entry contains ≥4 standalone "I didn't know that" facts as bullets, so content-writer can lift them directly.
  6. Register. EN body leans technical/analytical (the engineer-parent). No ZH ever (ADR-010).
  7. Filename = slug = ID. Kebab-case; the slug is what other entries reference in related:.
  8. Disclaimer blockquote. Mirror corpus/proof/epf-shortfall-cases.md: a leading > block stating what's sourced, what's illustrative, and that nothing is personalised advice.
⚠ Data dependency Output fidelity jumps a lot if Cluster A and the fund entries can cite real numbers. If you have the FundMaster Excel (fund-screener skill) or the current PMB fee schedule / prospectus, supplying it first replaces flagged placeholders with real figures. Otherwise entries ship with figures marked "consultant to confirm."

Task 1 Primitives

The transverse layer that unlocks every cluster. Front-loaded.

  • glossary.md — ≥12 investor terms (NAV, unit, bid-offer, sales charge, switching fee, redemption, distribution vs dividend, reinvestment, mgmt fee, trustee fee, ex-date, T+ settlement), each tied to how it shows on a PMB statement. ≥4 carry a non-obvious clause.
  • compliance-claim-phrasebank.md — two lists: Claimable (past-performance disclaimer, how to cite a factsheet figure) and Banned (guaranteed returns, "risk-free", capital-protection implications). Cross-refs corpus/compliance/public-mutual.md. This is what keeps anger/fear-register hooks compliant.
  • proof-asset-index.md — registry table of available proof assets (factsheets, prospectus clauses, calculator outputs, data files) → what claim each can back. The map brief-writer uses to pick a proof type per Brief.
